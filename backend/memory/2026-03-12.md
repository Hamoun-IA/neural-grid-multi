API REST + WebSocket backend pour le dashboard Neural Grid.

**Workspace** : `/root/.openclaw/workspace-backend/`  
**Port** : 3101  
**Stack** : Express + TypeScript + ws

## Structure

```
src/
  index.ts           — Express + WebSocket server
  config.ts          — Config des 5 serveurs (tokens depuis .env)
  types.ts           — Interfaces TypeScript
  routes/
    health.ts        — GET /api/health
    servers.ts       — GET /api/servers, GET /api/servers/:id
    webhook.ts       — POST /api/webhook/activity (Activity Reporter)
  services/
    gateway.ts       — Interrogation des gateways OpenClaw
    poller.ts        — Polling toutes les 60s + WebSocket broadcast + updateCachedAgentActivity()
scripts/
  activity-reporter.sh      — Script à déployer sur chaque serveur
  activity-reporter.service — Template systemd
```

## API Gateway OpenClaw — découvertes

L'API REST des gateways expose :
- `GET /health` → `{"ok":true,"status":"live"}` — health check simple
- `POST /tools/invoke` → invoke un tool, utilisé pour `sessions_list`

La plupart des paths (`/api/config`, `/api/sessions`) renvoient 404.  
Les paths `/agents`, `/status`, etc. renvoient l'UI HTML (SPA fallback).

**`sessions_list` via tools/invoke** retourne :
```json
{
  "count": 5,
  "sessions": [
    { "key": "agent:<agentId>:<channel>:...", "model": "...", "updatedAt": 12345 }
  ]
}
```

Les agent IDs sont parsés depuis le `key` avec regex `^agent:([^:]+):`.

## Mise à jour 2026-03-12 ~10:48 UTC — Webhook + Activity Reporter

### Nouveau : endpoint webhook (temps réel)

**Route** : `POST /api/webhook/activity`  
Appelée par le script `activity-reporter.sh` tournant sur chaque serveur.

**Body** :
```json
{
  "serverId": "NOVA",
  "serverName": "Nova",
  "agents": [
    { "id": "main", "status": "ACTIVE", "lastAge": "just now" },
    { "id": "painter", "status": "IDLE", "lastAge": "1h ago" }
  ],
  "timestamp": "2026-03-12T10:45:00Z"
}
```

**Comportement** :
- Broadcast WebSocket immédiat → frontend se met à jour en temps réel
- Merge dans le cache `serverStates` via `updateCachedAgentActivity()`
- Retourne `{ ok: true, received: N }`

### Nouveau : `updateCachedAgentActivity()` dans poller.ts

Fusionne les agents reçus par webhook dans le cache sans refaire de poll SSH.
Si le serveur est inconnu, crée une entrée minimale.

### Polling SSH réduit à 60s (backup only)

`POLL_INTERVAL_MS = 60_000` — les webhooks gèrent le temps réel, le polling SSH est un fallback.

### Scripts créés

- `scripts/activity-reporter.sh` — à déployer sur chaque serveur, lit `openclaw status`, pousse les changements au Hub via webhook
- `scripts/activity-reporter.service` — template systemd (copier dans `/etc/systemd/system/activity-reporter@.service`)

**Déploiement sur un serveur** :
```bash
sudo cp scripts/activity-reporter.sh /opt/neural-grid/
sudo chmod +x /opt/neural-grid/activity-reporter.sh
sudo cp scripts/activity-reporter.service /etc/systemd/system/activity-reporter@.service
sudo systemctl enable --now activity-reporter@NOVA
```
